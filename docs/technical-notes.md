# Technical Notes

READMEを短く保つため、実装上の補足をここにまとめます。

## 検索

人物、コミュニティ、話題、会話記録から検索用ドキュメントを作成し、検索用テキストとembeddingを保存します。

`OPENAI_API_KEY` が設定されている場合はOpenAI Embeddingsを利用します。未設定の場合は、外部サービスなしでデモを動かせるようにローカルのhash embeddingへフォールバックします。

検索結果は以下のスコアを組み合わせて並べます。

```text
0.55 * semantic_score
+ 0.35 * keyword_score
+ 0.10 * recency_score
```

初回検索時には `search_documents` からメモリ上の検索キャッシュを作成し、以後の検索で同じドキュメント読み込みやembedding JSONのパースを繰り返さないようにしています。

## 共有度

各やり取りには `share_level` を持たせています。

| Value | Meaning |
| --- | --- |
| `SHARED` | すでに話した |
| `PARTIAL` | 一部だけ話した |
| `WITHHELD` | まだ話していない |

人物ダッシュボードでは、この共有度を使って「次に自然に話せそうなこと」と「まだ直接触れない方がよいこと」を分けて確認できます。

## アカウントスコープ

DBスキーマには `accounts` と各主要テーブルの `account_id` があります。現時点ではローカルデモ用の固定デフォルトアカウントを使っていますが、後から認証を追加できる構造にしています。

## 現在の制約

- 認証は未実装で、固定のデフォルトアカウントを使っています。
- AIによるinsight生成、関係性更新、カレンダー同期は本番機能としては未実装です。
- 大規模データでは、pgvectorやPostgreSQL全文検索などDB側の検索最適化が必要です。
- フロントエンドのE2Eテストはまだありません。
- Docker Composeはまだありません。
