# test_api_smoke.py

## 概要

主要APIを一通り実行し、基本的な参照データ作成、インタラクション記録、検索、人物ダッシュボード、概要取得が動作することを確認するスモークテストです。

## テスト内容

### `test_01_health`

ヘルスチェックAPIが200を返し、レスポンスが `{"status": "ok"}` になることを確認します。

### `test_02_reference_endpoints`

人物、コミュニティ、トピックの一覧APIが200を返し、事前に作成したデータが一覧に含まれることを確認します。

### `test_03_duplicate_community_sibling_is_rejected`

同じ親配下に重複するコミュニティ名を作成しようとした場合、409で拒否されることを確認します。

### `test_03_manage_visibility_and_delete`

一時的な人物とコミュニティを作成し、非表示設定、非表示込み一覧での取得、削除後の一覧非表示を確認します。

### `test_04_record_interaction`

人物、コミュニティ、トピックに紐づくインタラクションを記録し、成功レスポンスと作成IDが返ることを確認します。

### `test_05_search_interactions`

記録済みインタラクションを条件検索できること、件数制限とページング情報付きレスポンスが動作することを確認します。

### `test_06_memory_search_endpoint`

検索APIが結果、グループ、回答サマリーを返し、対象種別フィルターで人物検索もできることを確認します。

### `test_07_person_dashboard`

人物ダッシュボードに概要、上位トピック、上位コミュニティ、会話準備情報が含まれることを確認します。

### `test_08_interaction_overview`

インタラクション概要APIが総件数、最近のインタラクション、人物別件数を返し、指定した上限内に収まることを確認します。

## 補助処理

### `setUpClass`

スモークテスト用の共通クライアントと参照データを作成し、後続テストで使うIDを保持します。

### `tearDownClass`

スモークテスト用に作成したデータを削除し、TestClientを閉じます。
